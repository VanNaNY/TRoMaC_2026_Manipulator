## 1. 雷达解析电磁波机制（规则手册侧）

## 1.1 机制定义与比赛作用

- 双方雷达基座放置官方发射源，发射包含**信息波**与**干扰波**的电磁信号。
- 红方基座发射含蓝方信息，蓝方基座发射含红方信息（对称）。
- 队伍可自制并安装天线/解调电路，在雷达端解析信息波与干扰波。
- 解析结果可用于：
  - 获取对手关键态势信息；
  - 通过密钥机制提升对方干扰难度，抑制对方雷达易伤能力。

## 1.2 信息波承载内容（规则定义）

信息波包含对方：

- 各机器人位置
- 各机器人血量
- 各机器人剩余发弹量
- 各机器人当前持有增益
- 队伍剩余经济
- 队伍总经济
- 各增益点被占领状态

对应通信命令在通信协议中映射为：

- `0x0A01` 位置
- `0x0A02` 血量
- `0x0A03` 发弹量
- `0x0A04` 宏观状态（经济、占领态势等）
- `0x0A05` 增益效果

## 1.3 干扰波承载内容（规则定义）

- 干扰波不承载完整业务信息。
- 干扰波包含一个**6位数字/字母组合密钥**（ASCII 字符）。
- 开局及干扰波难度变化时，密钥重置为随机值。
- 默认开局双方干扰波难度均为一级。

## 1.4 密钥破解与加密等级联动

- 雷达可将解析出的对方密钥发给赛事服务器验证。
- 验证成功后，对方干扰波解析难度提升 1 级（对应“己方加密等级”上升）。
- 当对方干扰波难度高于己方时，对方雷达即使标记准确，也不会产生易伤或累积“双倍易伤”进度。

## 2. 物理层与空口层参数（规则手册附录1）

## 2.1 调制方式

- 调制：`2-GFSK`（二进制高斯频移键控）
- FSK 映射：`0 -> -1`，`1 -> +1`
- BT：`0.35`
- Sample Rate：`1M`
- SPS：`52`

## 2.2 广播源/干扰源参数（关键值）

### 红方相关波源

- 红方广播源：`433.2 MHz`，带宽 `0.54 MHz`，功率 `-60 dBm`
- 红方一级干扰：`432.2 MHz`，带宽 `0.94 MHz`，功率 `-10 dBm`
- 红方二级干扰：`432.5 MHz`，带宽 `0.86 MHz`，功率 `-10 dBm`
- 红方三级干扰：`432.8 MHz`，带宽 `0.25 MHz`，功率 `-10 dBm`

### 蓝方相关波源

- 蓝方广播源：`433.92 MHz`，带宽 `0.54 MHz`，功率 `-60 dBm`
- 蓝方一级干扰：`434.92 MHz`，带宽 `0.94 MHz`，功率 `-10 dBm`
- 蓝方二级干扰：`434.62 MHz`，带宽 `0.86 MHz`，功率 `-10 dBm`
- 蓝方三级干扰：`434.32 MHz`，带宽 `0.25 MHz`，功率 `-10 dBm`

## 2.3 空口帧结构

每个空口包：

- `Access Code`：8字节（64bit）
  - 信息波：`0x2F6F4C74B914492E`
  - 干扰波：`0x16E8D377151C712D`
- `Header`：4字节
  - 两份 16-bit Payload Length（大端），固定值 `0x000F`
  - 两份不一致则丢包
- `Payload`：15字节

补充：

- 红蓝双方信息波 Access Code 相同，干扰波 Access Code 也相同。
- 原始数据流按 15 字节分片，逐包发送。
- 无独立前导码、无固定包间隔、无 Whitening、无 FEC。

## 2.4 时域特征（示例：红方广播源）

- 每 bit 时长：`52 us`
- 每包时长：`11.23 ms`（216 bits）
- bit=1 瞬时频偏：`+250769 Hz`
- bit=0 瞬时频偏：`-250769 Hz`

## 2.5 推送速率

- 信息波：`1350 byte/s`，全部为有效数据（`0x0A01~0x0A05`，10Hz）
- 干扰波：`1350 byte/s`，包含 `0x0A06`（10Hz）+ 其余随机填充

## 3. 雷达无线链路协议（通信协议侧）

## 3.1 链路定位

- 裁判系统串口体系包含三类链路：
  - 常规链路
  - 图传链路
  - **雷达无线链路**
- 雷达无线链路方向：`信号发射源 -> 雷达`
- 命令集合：`0x0A01`~`0x0A06`

## 3.2 命令字语义总览

- `0x0A01`（24字节）：对方机器人位置坐标（英雄/工程/步兵3/步兵4/空中/哨兵，x,y，单位 cm）
- `0x0A02`（12字节）：对方机器人血量（英雄/工程/步兵3/步兵4/保留/哨兵）
- `0x0A03`（10字节）：对方允许发弹量（英雄/步兵3/步兵4/空中/哨兵）
- `0x0A04`（8字节）：对方宏观状态（剩余金币、累计金币、占领与关键点状态 bitfield）
- `0x0A05`（36字节）：对方机器人增益（回血/冷却/防御/易伤/攻击等，分机器人打包）
- `0x0A06`（6字节）：对方干扰波密钥（6字节 ASCII 数字字母）

## 3.3 重点字段说明

### `0x0A04`（宏观状态）

- 包含经济信息：`remaining_gold`、`total_gold`
- 包含战场占领态势 bit 位：补给区、中央高地、梯形高地、堡垒、前哨站、基地、隧道/公路/飞坡等关键点位状态
- 这是雷达端推演对方节奏与资源状态的核心字段之一。

### `0x0A05`（增益状态）

- 对方各机器人分别给出：
  - 回血增益
  - 热量冷却增益（直接值）
  - 防御增益
  - 负防御增益（易伤）
  - 攻击增益
- 哨兵额外含当前姿态字段（进攻/防御/移动）。

### `0x0A06`（密钥）

- 精确定义为 6 字节 ASCII 字母或数字。
- 与规则手册中的“干扰波密钥破解机制”直接对应。

## 4. 雷达主动交互命令（与解析波机制耦合）

雷达可通过常规链路向服务器发送自主决策子命令：

- 子命令：`0x0121`（雷达自主决策指令）
- 关键能力：
  - 请求触发“双倍易伤”
  - 上传破解到的对方密钥进行验证

`0x0121` 字段中的密钥指令规则：

- `byte1=1`：更新己方密钥
- `byte1=2`：上传破解到的对方密钥进行校验
- `byte2~7`：6位密钥 ASCII
- 校验密钥后 10 秒内再次更新无效

## 6. 工程侧注意事项（从规则反推）

- 规则与协议都强调：雷达信息波数据结构以通信协议为准，建议优先按 `0x0A01~0x0A06` 建模。
- 干扰波密钥是机制闭环核心（解析 -> 上传验证 -> 对方难度提升 -> 对方易伤能力受限）。
- 由于比赛环境存在延迟、丢包与坐标误差，策略层应设计：
  - 缓冲与去抖；
  - 多帧确认；
  - 超时回退。
- 0x0A04/0x0A05 对战术价值高，建议在雷达端建立“状态估计缓存”并加时间戳。
