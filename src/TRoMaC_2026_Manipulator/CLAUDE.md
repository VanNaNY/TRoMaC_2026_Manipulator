## Build & Run

The workspace root is `/home/tromac/moveit2_ws`. Always build from there:

```bash
# Build this package only
cd ~/moveit2_ws
colcon build --packages-select TRoMaC_2026_Manipulator

# Source before running anything
source ~/moveit2_ws/install/setup.bash
```

There are no tests in this package. No `ament_lint` or test targets are configured in CMakeLists.txt.

### Launch Files

```bash
# Full demo with RViz (motion planning via MoveIt)
ros2 launch TRoMaC_2026_Manipulator demo.launch.py

# Lower-computer joystick → MoveIt Servo (primary control mode)
ros2 launch TRoMaC_2026_Manipulator servo_serial.launch.py
ros2 launch TRoMaC_2026_Manipulator servo_serial.launch.py serial_device:=/dev/ttyUSB0

# MoveIt Servo + RViz + keyboard teleoperation (debug / fallback)
ros2 launch TRoMaC_2026_Manipulator servo_keyboard.launch.py
ros2 launch TRoMaC_2026_Manipulator servo_keyboard.launch.py start_keyboard:=false

# Serial bridge only (TX joint angles, no servo control)
ros2 run TRoMaC_2026_Manipulator serial_comm_node \
  --ros-args -p device:=/dev/ttyACM0 -p baud_rate:=921600 -p enable_servo_control:=false
```

## Architecture

This is a **MoveIt 2 configuration package** generated by MoveIt Setup Assistant, extended with two custom C++ nodes.

### Robot Model

- URDF source lives in the separate **`manipulator`** package (`urdf/manipulator.urdf`); this package only references it via `.setup_assistant` and the xacro files in `config/`.
- 6-DOF arm with joints: `yaw-1-joint`, `pitch-1-joint`, `pitch-2-joint`, `roll-1-joint`, `pitch-3-joint`, `roll-2-joint`
- End-effector frame: `roll-2_Link`; planning group name: `manipulator`
- IK solver: **TRAC_IK** (`config/kinematics.yaml`)
- `config/joint_limits.yaml`: velocity/acceleration limits are currently disabled (all set to false/0) — only the URDF limits apply.

### Control Stack

```
MoveIt move_group ──► JointTrajectoryController (ros2_control, 100 Hz)
                              │
                              ▼ (position commands)
                     serial_comm_node ──► UART /dev/ttyACM0 @ 921600 baud
```

MoveIt Servo (`moveit_servo`) provides real-time Cartesian/joint jogging as an alternative path to the same `JointTrajectoryController`.

### Custom Nodes (`src/`)

**`serial_comm_node`** (`serial_comm_node.cpp` + `Serial.h`/`Serial.cpp`):
- Bridges ROS 2 to the physical MCU over UART
- **TX frame** (12 bytes): 6 × `int16_t` joint angles in integer degrees → `VisionFrameTX_structTypedef`
- **RX frame** (12 bytes): arm position x/y/z, pitch, roll, button, end-byte → `VisionFrameRX_structTypedef`
- Subscribes to `serial_send` (`std_msgs/Float64MultiArray` with 6 doubles) for on-demand TX sends
- A background thread reads incoming frames using `select()` with 300 ms timeout
- **Servo control** (`enable_servo_control`, default true): reads RX joystick data and publishes `geometry_msgs/TwistStamped` to `/servo_node/delta_twist_cmds` at 50 Hz
  - Axis mapping (base_link frame): `Arm_Pos_x/y/z` [-1000,1000] → `linear.x/y/z`; `Arm_Pos_Roll/Pitch` [-100,100] → `angular.x/y`; Yaw=0
  - Dead-zone applied before scaling: `joy_deadzone_xyz`=30, `joy_deadzone_angular`=3
  - Speed cap: `joy_max_linear`=0.5, `joy_max_angular`=0.5 (combined with servo_settings.yaml scale)
  - Publishes zero twist if no RX frame within `rx_timeout_ms`=200 ms (arm stops safely)
  - Calls `/servo_node/start_servo` service in a detached thread at startup
- TX parameters: `device`, `baud_rate`, `auto_send`, `send_rate_hz`, `tx_joints`, `send_once_on_start`, `log_auto_send`

**`servo_keyboard_input`** (`servo_keyboard_input.cpp`):
- Keyboard teleoperation for MoveIt Servo
- Calls `/servo_node/start_servo` service on startup (waits up to 15 s)
- Publishes `geometry_msgs/TwistStamped` → `/servo_node/delta_twist_cmds` for Cartesian jog
- Publishes `control_msgs/JointJog` → `/servo_node/delta_joint_cmds` for per-joint jog
- Opens `/dev/tty` as fallback when stdin is not a terminal (ros2 launch case); if no TTY is available it throws and exits cleanly

### Key Config Files

| File | Purpose |
|------|---------|
| `config/ros2_controllers.yaml` | ros2_control: `manipulator_controller` (JointTrajectoryController) + `joint_state_broadcaster` |
| `config/moveit_controllers.yaml` | MoveIt controller manager mapping to `manipulator_controller` |
| `config/servo_settings.yaml` | Servo: unitless commands, 0.01 s period, Butterworth smoothing, collision checking enabled |
| `config/kinematics.yaml` | TRAC_IK, 5 ms timeout |
| `config/pilz_cartesian_limits.yaml` | Pilz planner Cartesian velocity/acceleration limits |
| `config/*.xacro` | Robot description with ros2_control hardware interface |
| `config/*.srdf` | MoveIt semantic description (planning groups, named states) |

### Launch Graph

`servo_keyboard.launch.py` composes:
1. `demo.launch.py` → which uses `MoveItConfigsBuilder` to bring up `move_group`, `robot_state_publisher`, `ros2_control`, RViz
2. `moveit_servo/servo_node_main`
3. `servo_keyboard_input` (conditional on `start_keyboard` arg)

All launch files load robot configuration via `MoveItConfigsBuilder("机械臂-总装-新-4.SLDASM", package_name="TRoMaC_2026_Manipulator")`.
